### Обязательные задания
1. Дисковые метрики - что бы следить за доступностью и свободным местом на них  
   CPU (+RAM) метрики - что бы следить за нагрузкой  
   HTTP метрики (RPS, процент неуспешных запросов, детализация по неуспешным) - что бы следить за трафиком и успешностью запросов
2. На метриках добавить подсказки или более подробное описание (например с указанием влияния)  
   Внедрить графики с метриками SLA / SLO / SLI
3. Sentry (или любой другой бесплатный аналог)? Но если в "не выделили финансирование" входит и железо, то сложно будет предоставить полноценное и централизованное решение
4. Значит около 30% это 1xx и 3xx коды ответов
5. #### Плюсы pull-модели:
- Легче контролировать подлинность данных
- Можно настроить единый proxy server до всех агентов с TLS
- Упрощённая отладка получения данных с агентов
  #### Минусы pull-модели:
- Если сервер приляжет, то данные монитринга не будут собираться ни откуда
- Хуже работает с динамическими средами (может не успеть собрать метрики)
   #### Плюсы push-модели:
- Упрощение репликации данных в разные системы мониторинга или их резервные копии
- Более гибкая настройка отправки пакетов данных с метриками
- UDP — это менее затратный способ передачи данных, из-за чего может возрасти производительность сбора метрик
   #### Минусы push-модели:
- Более сложные настройки по контролю трафика (может быть большая единовременная нагрузка с нескольких агентов)
