### Задача 1: Кластеризация
Прелагаю использовать Kubernetes (k8s)\
Он соответствует всем заданным требованиям\
Для обеспечения большей безопасности возможно использование Kubernetes совместно с HashiCorp Vault\
Более того, Kubernetes можно считать устаявшимся стандартом (практически монополист в этой сфере и для этих требований)

#### Репликация:
- Используем контроллер Deployment: мы указываем количество реплик (например `replicas: 3`) и контроллер следит, чтобы в кластере всегда было запущено именно столько экземпляров приложения (если один под упадет k8s автоматически поднимет замену)

#### Балансировка трафика:
- Внутри кластера: используем абстракцию Service (тип ClusterIP), которая дает единый стабильный IP-адрес для группы подов и распределяет запросы между ними
- Снаружи: для доступа из интернета ставим Ingress-контроллер (например Nginx), который принимает внешний трафик и маршрутизирует его по нужным сервисам на основе доменных имён или путей

#### Масштабирование:
- Настраиваем Horizontal Pod Autoscaler, если нагрузка на CPU или RAM начнёт расти, k8s автоматически увеличит количество реплик

#### Хранение чувствительных данных:
- Используем Secrets - пароли, ключи и токены не хранятся в коде или Docker-образе, они создаются в k8s и пробрасываются в приложение как переменные окружения
- Для большей безопасности можно интегрировать HashiCorp Vault

#### Процесс деплоя и развертывания сервисов приложения на кластер k8s:
- GitLab CI - для сборки и тестирования
- Container Registry - для хранения Docker-образов
- Kubectl - для деплоя
