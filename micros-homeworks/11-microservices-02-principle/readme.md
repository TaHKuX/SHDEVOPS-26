### Задача 1: API Gateway
| Решение | Конфигурация | Аутентификация | HTTPS |
| - | - | - | - |
| Kong (OSS) | Динамическая (через API или YAML) | JWT, Key Auth и OAuth2 | Да |
| KrakenD | Статическая (JSON/YAML) | JWT и RBAC | Да |
| Tyk (OSS) | Динамическая (Dashboard/файлы) | OIDC, OAuth2 и API-ключи | Да |
| NGINX (OSS) | Статическая (через `.conf` файлы) | Модули или Lua-скрипты | Да |
| AWS API Gateway | Визуальная / Swagger / IaC | Cognito и Lambda | Да |

Выберу Kong, т.к. есть плагины для авторизации и можно менять конфигурацию через API
### Задача 2: Брокер сообщений
| Решение | Кластеризация | Хранение на диске | Скорость работы | Форматы сообщений | Права доступа | Простота эксплуатации |
| - | - | - | - | - | - | - |
| RabbitMQ | Да (Quorum Queues / Raft) | Да (Persistent messages) | Средняя | Любые | Гибкие (Vhost, Exchange, Queue) | Высокая (UI, плагины) |
| Apache Kafka | Да (Высокая избыточность) | Да (Commit Log) | Самая высокая | Любые | Продвинутые (Topics/Groups) | Низкая (сложный конфиг) |
| Redis Streams | Да (Sentinel / Cluster) | Опционально (AOF/RDB) | Очень высокая | Любые | Базовые (ACL по командам) | Очень высокая |
| NATS JetStream | Да (Raft-консенсус) | Да | Высокая | Любые | Гибкие (Subject-based) | Средняя |

Выберу RabbitMQ, т.к. соответствует всем требованиям, проверен годами и достаточно прост т.к. имеет UI и плагины
